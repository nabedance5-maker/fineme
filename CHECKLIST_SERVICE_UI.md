検証チェックリスト — Service / Store UI

目的: タブ統合・インラインサービス表示・管理画面ギャラリー並べ替え・カード表示のレスポンシブを確認する

前提: 開発サーバーまたは静的ファイルをブラウザで開ける状態

1) 店舗ページのタブと deep-link
- URL 例: `store.html?providerId=<providerId>&tab=menu&serviceId=<serviceId>` を開く
- 確認事項:
  - ページが開き、タブが表示される
  - `tab=menu` と `serviceId` があると、該当サービスが自動で展開される
  - タブをキーボードで操作（Tab でフォーカス→ArrowLeft/ArrowRight/Home/End で移動）できる
  - パネル要素は `role=tabpanel` と `aria-labelledby` を持つ

2) インラインサービス詳細のアクセシビリティ
- 展開直後、該当サービスの領域にフォーカスが移る
- スクリーンリーダー用のライブリージョンで短い案内が読み上げられる（"サービス『〜』を開きました。"）
- 予約ボタンに `aria-label` が付与されている

3) サービスカード (トップ / 検索 / React ServiceCard)
- カードに価格バッジが表示される
- 店舗名はストア deep-link に遷移する
- 予約ボタンは視覚的に目立つ（グラデーション）
- レスポンシブ: ウィンドウ幅を変え、カードの列が自動的に詰まること（260px を下回ると 1 列）

4) 管理画面 - ギャラリー並べ替え
- プロバイダで管理画面を開き、サービス編集モーダルを開く
- ギャラリーに複数画像がある状態にする
- 画像をドラッグ＆ドロップして順序を入れ替えられる
- 保存後に再度編集し、順序が保持されていることを確認する

5) レビュー表示・案内
- サービス詳細内のレビュー欄に "レビューの投稿はマイページ › 予約履歴から行えます" の案内が表示される
- 通報ボタンを押すと通報ダイアログ（prompt）が出て処理される

トラブルシューティング
- 画像が表示されない: `service.photo` のパスや placeholder パスを確認
- ローカルStorage が古い場合: `localStorage.clear()` 後にダミーデータを再投入

完了したら: 確認結果（OK / NG）と発生した不具合のスクリーンショットや console エラーを共有してください。
